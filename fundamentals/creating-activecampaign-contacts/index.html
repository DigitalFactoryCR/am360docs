Creating ActiveCampaign Contacts

A contact within your primary CRM can be created in many ways; manually, using the features of the primary CRM, by MembersPlus or by other third party plugins and applications.

There are numerous applications, SaaS applications and third party WordPress plugins, that can create a contact in your primary CRM.

They could be services such as Zapier and Integromat. They could be form submission products such as ConvertBox, Convertful or WordPress plugins such as Gravity Forms etc. How they function is not explained here as it is outside the scope of this documentation. However you should be aware that there are numerous ways to create a contact within the primary CRM, without using your primary CRM directly or the features of MembersPlus.

Typically using the features of the primary CRM a contact can be created manually, when using a form defined in the primary CRM, or when imported.

The main options for creating a primary CRM contact using the features of MembersPlus are: Using the MembersPlus module â€“ Auto-Login

If you are using an external cart (i.e. one not hosted in WordPress) MembersPlus provides a module that allows validated purchasers to autologin to the WordPress MembersPlus site using an autologin URL as the Success or Thank You Page URL for the purchase.

This is the MembersPlus Auto-Login module.

The Auto-Login module when used in conjunction with an external cart such as; ThriveCart, Digistore24, ClickBank or JVZoo will automatically create a contact within the primary CRM if the order credentials passed to MembersPlus validate successfully and the corresponding contact, i.e. one with a matching email address, does not already exist.

Details of how to implement this module are available at MembersPlus, Modules, Auto-Login.

Furthermore if the MembersPlus Auto-Login module also creates the WordPress user and the MembersPlus, Module, User Registration is enabled the settings there will also apply, to add specific primary CRM tags and set a contact field for the new or existing primary CRM contact. Using the MembersPlus shortcode â€“ [mbr_form_register]

MembersPlus provides the shortcode [mbr_form_register] which can be used on any post/page to allow a visitor to register on the WordPress site.

When using this shortcode the user is always created as a contact in your primary CRM if the corresponding contact, i.e. one with a matching email address, does not already exist.

If the MembersPlus, Module, User Registration is enabled the settings there will also apply, to add specific primary CRM tags and set a contact field for the new or existing contact.

Details of how to use this form are available at MembersPlus Shortcode [mbr_form_register]. WordPress Administrator User Functions

A WordPress Administrator can manually create WordPress users using the WordPress administrator function WordPress, Users, Add New.

If the MembersPlus, Module, User Registration is enabled they will also be added as contacts to the primary CRM if the corresponding contact, i.e. one with a matching email address, does not already exist and the settings for MembersPlus, Module, User Registration apply. The new or existing contact will have primary CRM tags added and a contact field set as appropriate. WordPress Registration

WordPress comes with built-in functionality to allow site visitors to register as WordPress users.

When users are created using this method if the MembersPlus, Module, User Registration is enabled they will also be added as contacts to the primary CRM if the corresponding contact, i.e. one with a matching email address, does not already exist and the settings for MembersPlus, Module, User Registration apply. The new or existing contact will have primary CRM tags added and a contact field set as appropriate. WooCommerce

A notable example of a third party WordPress plugin that can create primary CRM contacts when using MembersPlus is WooCommerce.

When using WooCommerce users can be created in WordPress using the WooCommerce, Settings, Accounts & Privacy, Account Creation features.

When users are created using this method if the MembersPlus, Module, User Registration is enabled they will also be added as contacts to the primary CRM if the corresponding contact, i.e. one with a matching email address, does not already exist and the settings for MembersPlus, Module, User Registration apply. The new or existing contact will have primary CRM tags added and a contact field set as appropriate.

Also, when using the MembersPlus WooCommerce Extension this can also create or update a contact within the primary CRM with the addition of specified tags etc dependent upon the visitors actions within WooCommerce.

Details regarding the extension are available at MembersPlus, Extensions, WooCommerce.
