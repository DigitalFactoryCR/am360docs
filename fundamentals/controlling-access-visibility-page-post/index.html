Protection Settings - Controlling Access To & Visibility Of Pages & Posts

Access to the entire page/post is controlled using the settings within the ActiveMember360 panel when creating or editing the page/post. These settings are very comprehensive and permit many combinations of access permissions to be defined.

The settings within the ActiveMember360 panel are also used to determine the behaviour the visitor experiences when they do not have the necessary permissions to view the page/post. This includes whether the page/post is shown in menus/widgets, the display of excerpts and any redirect performed.

Page/Post Protection Using ActiveMember360 Panel#

The available options and permission settings are:

Inherit from parent#

This setting will allow access to anyone who meets the access requirements for this pageâ€™s parent page.

Note

This option is not available for protecting posts.

Anyone at anytime#

This setting will allow access to everyone at all times irrespective of whether a Force Login, as defined within ActiveMember360, Plugin Options, Login Settings, Always require login is required.

An obvious use for this is to provide everyone access to your privacy policy, terms of use etc.

Any logged in user#

This setting will grant access to the page/post for any logged in user irrespective of their memberships.

Memberships#

These settings will grant access to the page/post for a logged in user dependent upon their memberships.

If a membership is selected access is allowed for any logged in user who belongs to that membership (i.e. has the ActiveCampaign tag that defines the selected membership), but always dependent upon other settings within the ActiveMember360 Protection panel.

If multiple memberships are selected access is allowed for any logged in user who belongs to at least one of the memberships (i.e. has at least one of the ActiveCampaign tags that define the selected memberships), but always dependent upon other settings within the ActiveMember360 Protection panel.

If no memberships are selected, protection will be determined entirely by the other settings specified within ActiveMember360 Protection panel.

Hide from menu only#

This setting will prevent this page/post from appearing in the menu/widget. It will not prevent access to the page/post itself.

This setting will not hide a menu/widget item defined as a custom link using the page/post URL.

However it does override any conditional test for this page/post within a menu itemâ€™s ActiveMember360 Menu Visibility settings or within a widgets ActiveMember360 Widget Control settings.

Hide completely (incl. excerpt)#

This setting will prevent this page/post from appearing in the menu/widget as well as deny access to anyone other than WordPress administrators.

This setting will not hide a menu/widget item defined as a custom link using the page/post URL.

However it does override any conditional test for this page/post within a menu itemâ€™s ActiveMember360 Menu Visibility settings or within a widgets ActiveMember360 Widget Control settings.

Show only to public visitors#

This setting will deny access to this page/post to anyone who is logged in. It will only be available to visitors who are NOT logged in.

Excerpt only with permissions#

This setting will result in excerpts being ignored when determining this page/postâ€™s visibility to a user unless a user first meets all permission requirements.

An excerpt will only be displayed to users who fulfil all permission requirements. Users who do not will see neither the full content nor the excerpt, if one exists.

Be aware that the visibility of the excerpt influences the visibility of pages/posts within menus and widgets.

Only/Not these Tags#

This is used to define additional tag criteria beyond basic membership tags.

Terms that define the criteria are entered using the dynamic picker. As you type a filtered list will be displayed. The displayed list is limited to 10 items even when the filter determines more. Terms are shown with the nomenclature TagId â€“ Tag Name that represents 'Has the tag named TagName'. For every tag there is also an additional term generated with the nomenclature not TagId â€“ Tag Name that represents 'Does not have the tag named TagName'. Clicking on the appropriate term adds it to the list. Incorrect or no longer required selections can be removed by clicking on the X.

As an example: By selecting; 123 â€“ TagNameFor123, 234 â€“ TagNameFor234 , 345 â€“ TagNameFor345, not 456 â€“ TagNameFor456, anyone who does not have the tag named TagNameFor456 and has either tag named TagNameFor123 or tag named TagNameFor234 or tag named TagNameFor345 is granted access to the page/post providing they meet any defined membership requirements.

When forbidden redirect to#

This setting is used to determine how a HTTP standard response '404 - Not Found' error is treated. This error typically arises when any of the following apply:

  • The user does not have permission to access a page/post and no excerpt is defined or being automatically generated for that page/post.
  • The requested page does not exist within the site.

The choice of settings are to redirect to:

  • The default 404 site handler.
  • Any of the pages defined in your site.
  • A custom URL.

Permissions#

If a user doesn't have the necessary permissions to view a page or post, the following happens in this order when that user attempts to access the page/post:

  1. If there's a manual excerpt for the page/post, it will be displayed.
  2. If not but there's an excerpt defined by the use of the tag in the page/post content that will be displayed.
  3. If not but the option ActiveMember360, Plugin Options, Main Settings, Excerpts, Force excerpts at all times? is set to Yes, an excerpt will be displayed that is automatically generated from the page/post content.
  4. If not but the option for ActiveMember360 Protection, When forbidden, redirect to is set to anything but the Default 404 Page Handler the user will be redirected to the URL or page specified.
  5. If not but the option ActiveMember360, Plugin Options, Main Settings, Special Pages, Alternate '404 - Page Not Found' page is set the user will be redirected to that page.
  6. If not and your theme includes a template called 404.php, that template will be called and the web page that generates displayed.
  7. If not, your server's default 404 Page Not Found will be displayed.

Special Pages#

ActiveMember360 treats certain pages within your WordPress site as special pages.

These pages will always be accessible to any visitor to the site because they are essential to the functioning of the site.

For more information please visit Special Pages.

Page/Post Visibility In Menus & Widgets#

Typically if pages/posts are protected using page/post level ActiveMember360 Protection they will not be seen in WordPress menus or widgets.

However they will be seen irrespective of the ActiveMember360 Protection settings if:

  • The menu/widget item has been specified using a custom link.

Or providing for the ActiveMember360 Protection settings Hide completely (incl. excerpt) or Hide from menu only have not been selected and if:

  • An excerpt exists for the page/post. This includes automatic excerpts.

  • The page/post has any level of child post/page that is visible in the menu or widget.

  • ActiveMember360 Widget Control settings or ActiveMember360 Menu Visibility settings determine the post/page should be shown.

Excerpts#

Excerpts are used in certain circumstances when the visitor to a protected page/post does not have the necessary permissions to view the entire page/post.

To gain a fuller understanding of excerpts and how they are used with ActiveMember360 please visit Excerpts.
