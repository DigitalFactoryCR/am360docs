Login Settings

Always require login#

When Always require login is set to Yes your entire site is hidden to any visitor unless they are logged in and have the necessary permissions to see the visited content. The only exception to this is for any pages/posts whose ActiveMember360, Protection is set to Anyone at any time see Protection Settings.

Those pages/posts will be visible to any visitor irrespective of the Always require login setting.

When Always require login is set to Yes the visitor is shown a login page of your choice which is selected in ActiveMember360, Plugin Options, Special Pages, Select your login page. By default this page is set to the WordPress wp-login.php page.

Default: No

Tag used to ban a subscriber from site#

Select the ActiveMember360 tag a user must have to be banned from being able to login to your site.

Default: None

Allow login without a membership tag#

Allow a login by a visitor who does not have any ActiveCampaign tags used to define ActiveMember360 Membership Levels.

Membership Levels are defined in ActiveMember 360, Membership Levels

Default: No

When using WooCommerce we would recommend that Allow login without any Access Level tag(s) is set to Yes. It will allow users to login to the site to complete any failed orders or similar.

Name of "last login" custom field#

Select the ActiveCampaign contact field where you want to store the date of the last login. Only ActiveCampaign 'date' fields will appear in the dropdown.

Default: None

Automation to run on login#

Select the ActiveCampaign automation that you wish to run whenever a user logs in.

Please note the automation will only run for remote users i.e. a logged in user who also exists as a contact within ActiveCampaign. See

Default: None

Automation to run on logout#

Select the ActiveCampaign automation that you wish to run whenever a user logs out.

Please note the automation will only run for remote users i.e. a logged in user who also exists as a contact within ActiveCampaign. See

Please note that if a member simply closes the browser, it will not register as a logout. For this option to activate, the member must actively log out.

Default: None

Page redirect on first login#

Select the page to which users should be redirected after their first successful login.

The use of Page redirect on first login allows, as an example, terms and conditions to be shown which the user must agree to before proceeding to access the rest of the site.

Default: In order of priority:

  • Any redirect set using a login form or login form shortcode such as [mbr_form_login].
  • Any setting in Membership Levels, Login Redirect, applying to the current user.
  • Your site home page.
