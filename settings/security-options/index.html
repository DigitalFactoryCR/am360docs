Concurrent Login Module Overview

The MembersPlus Concurrent Login Module prevents concurrent logins from users with the same login credentials.

When a concurrent login occurs the user who is currently logged is authenticated and allowed access to the site.

However any other WordPress sessions currently active for that user will be terminated i.e. the user is effectively logged out.

A concurrent login can occur if the user logs in from a different browser window, and/or different device. It is not dependent upon IP address.

Also when a concurrent login occurs an email can be sent to an email address of choice, to notify the site administrator for example.

Activation & Configuration Activate Concurrent Login Module Steps to activate and configure the Concurrent Login Module

In WordPress admin sidebar click MembersPlus, Modules.
In MembersPlus Modules tabs click Concurrent Login.
Set Enable concurrent login to Yes.
To have an email sent set Email Notification to Yes.
The notification email is automatically scheduled to send every hour. After the first email further emails will only contain details of concurrent logins since the last notification email was sent.
If Email Notification is set to No concurrent login data will be available from within the WordPress database. It is stored within the WordPress options table in the record with the option_name mbr_concurrent_logins
When Email Notification is set to Yes specify the email address that should receive the concurrent login notification emails.
If this field is left empty the default WordPress admin email address will be used to populate this field automatically upon applying Save Settings. This is the address as defined in the WordPress admin sidebar Settings, General, Administration Email Address.
The details contained within the email includes for each concurrent login:
WordPress User ID
WordPress Username or Email Address
WordPress First Name
WordPress Last Name
Date and time of concurrent login using server time
IP Address
Click Save Settings.

Security Codes

Define as many Security Codes as required.

These are used with MembersPlus Webhooks for authorization purposes.

We recommend you use only the following characters and symbols alphanumerics [0-9a-zA-Z] and special characters \$-_.+!*(),

This will ensure that your codes are safe for use with all webhook URLs.
